phdi.fhir.transport.export module¶

phdi.fhir.transport.export.export_from_fhir_server(cred_manager, fhir_url, export_scope='', since='', resource_type='', container='', poll_step=30, poll_timeout=300)[source]¶

Initiate a FHIR $export operation, and poll until it completes. If the export operation is in progress at the end of poll_timeout, use the default polling behavior to do the last function check before shutting down the request.

Parameters

  • cred_manager (phdi.cloud.core.BaseCredentialManager) -- Service used to get an access token used to make a request

  • fhir_url (str) -- FHIR Server base URL

  • export_scope (str) -- Either Patient or Group/[id] as specified in the FHIR spec (https://hl7.org/fhir/uv/bulkdata/export/index.html#bulk-data-kick-off-request)

  • since (str) -- A FHIR instant (https://build.fhir.org/datatypes.html#instant) instructing the export to include only resources created or modified after the specified instant

  • resource_type (str) -- A comma-delimited list of FHIR resource types to include

  • container (str) -- The name of the Azure blob container used to store exported files

  • poll_step (float) -- The number of seconds to wait between poll requests, waiting for export files to be generated

  • poll_timeout (float) -- The maximum number of seconds to wait for export files to be generated

Return type

dict

phdi.fhir.transport.export.export_from_fhir_server_poll(poll_url, cred_manager, poll_step=30, poll_timeout=300)[source]¶

Poll an endpoint to retrieve an export file after an export run has been initiated.

Parameters

  • poll_url (str) -- URL to poll for export information

  • cred_manager (phdi.cloud.core.BaseCredentialManager) -- Service used to get an access token used to make a request

  • poll_step (float) -- the number of seconds to wait between poll requests, waiting for export files to be generated

  • poll_timeout (float) -- the maximum number of seconds to wait for export files to be generated

Raises

  • polling.TimeoutException -- If the FHIR server continually returns a 202 status indicating in progress until the timeout is reached

  • requests.HTTPError -- If an unexpected status code is returned

phdi.fhir.transport.http.http_request_with_reauth(cred_manager, url, retry_count, request_type, allowed_methods, headers, data=None)[source]¶

First, call utils.http_request_with_retry(). If the first call failed with an authorization error (HTTP status 401), obtain a new token using the cred_manager, and if the original request had an Authorization header, replace with the new token and re-initiate utils.http_request_with_retry().

Parameters

  • cred_manager (phdi.cloud.core.BaseCredentialManager) -- credential manager used to obtain a new token, if necessary

  • url (str) -- The url at which to make the HTTP request

  • retry_count (int) -- The number of times to re-try the request, if the first attempt fails

  • request_type (Literal['GET', 'POST']) -- The type of request to be made. Currently supports GET and POST.

  • allowed_methods (List[str]) -- The list of allowed HTTP request methods (i.e. POST, PUT, etc.) for the specific URL and query

  • headers (dict) -- JSON-type dictionary of headers to make the request with, including Authorization and content-type

  • data (Optional[dict]) -- JSON data in the case that the request requires data to be posted. Defaults to none.

Returns

A requests.Request object containing the response from the FHIR server.

Return type

requests.models.Response
