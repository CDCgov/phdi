phdi.transport.http module¶

phdi.transport.http.http_request_with_retry(url, retry_count, request_type, allowed_methods, headers, data=None)[source]¶

Execute an HTTP request, retrying the request if the returned HTTP status code is one of a specified list of codes.

Parameters

  • url (str) -- The url at which to make the HTTP request.

  • retry_count (int) -- The number of times to retry the request, if the first attempt fails.

  • request_type (Literal['GET', 'POST']) -- The type of request to be made. Currently supports GET and POST.

  • allowed_methods (List[str]) -- The list of allowed HTTP request methods (i.e. POST, PUT, etc.) for the specific URL and query.

  • headers (dict) -- JSON-type dictionary of headers to make the request with, including Authorization and content-type.

  • data (Optional[dict]) -- The data as a JSON-formatted dictionary, used when the request requires data to be posted. Default: None

Raises

ValueError -- An unsupported HTTP method (e.g. PATCH, DELETE, etc) was passed to the request_type parameter.

Returns

A HTTP request response.

Return type

requests.models.Response
